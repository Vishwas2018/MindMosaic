## 1. Context

**Position in sequence.** The polish stage runs after v1.1-S7 (Content Authoring &
Bank Population) closes and before the preview-environment deployment gate and full
Playwright E2E run. It is not a discovery pass — all input issues have been filed.
This stage consumes them.

**Input.**
- ADR-0040 audit findings: ISSUE-0039 through ISSUE-0048 (10 issues, filed 2026-05-18;
  ISSUE-0042 resolved 2026-05-19 at the pre-S6 dedicated remediation commit (b3eb668),
  per ADR-0040 §S6 blockers).
- S6/S7 carries: ISSUE-0049 (fuzzy/embedding duplicate detection), ISSUE-0050
  (cross-import exact-match dedup). Both are post-launch backend concerns with
  no v1.1 UI surface — out of scope for this stage; tracked in OPEN_ISSUES.md.

**What this stage is not.** This is not a free-form improvement pass. Every change
must trace to a filed issue or to a specific 5-state matrix gap identified by the
operator at sketch-gate. Untraced changes are out of scope.

## 2. In Scope

### 2a. Issue remediation (by severity)

**High — already closed.**
- ISSUE-0042 (Zod parse gap, content-svc + assessment-svc) — resolved 2026-05-19 at
  the pre-S6 dedicated remediation commit (b3eb668), per ADR-0040 §S6 blockers.

**Medium — full remediation required.**
- ISSUE-0039: Submit error does not discriminate 402 Upgrade Required.
  Files: `apps/web/src/components/student/StudentComposerForm.tsx:359–362`,
  `apps/web/src/app/(teacher)/content/new/page.tsx:347–354`.
  Fix: check `error.status === 402`; render `<UpgradeState />` or route to upgrade page.
- ISSUE-0040: SDK hooks missing `staleTime` causes refetch storms.
  Files: `packages/sdk/src/hooks/content.ts` (usePathways),
  `packages/sdk/src/hooks/session.ts` (useSessionState),
  `packages/sdk/src/hooks/assignments.ts` (useAssignmentsForClass).
  Fix: add appropriate `staleTime` per hook type; coordinate with cache invalidation.
- ISSUE-0041: N+1 query patterns in assignments-svc (4 await-in-loop sites).
  Files: `supabase/functions/assignments-svc/handlers.ts`
  (publishAssignment ~570–576, getAssignmentsForStudent ~716–720,
  getAssignmentsForClass ~784–787, getAssignmentTracking ~813–816).
  Fix: batch `fetchDisplayName` lookups into single `IN (...)` query per handler.
- ISSUE-0043: assessment-svc `/respond` and `/submit` missing Idempotency-Key enforcement.
  Files: `supabase/functions/assessment-svc/index.ts`.
  Fix: add Idempotency-Key header extraction + idempotency-window check (content-svc pattern).
- ISSUE-0045: Focus management missing on `/practice` and `/exam-sim` route entry.
  Files: `apps/web/src/app/(student)/practice/page.tsx`,
  `apps/web/src/app/(student)/exam-sim/page.tsx`.
  Fix: `useEffect(() => { headingRef.current?.focus() }, [])` on route entry.

### 2b. UI consistency pass

Scope: all v1.1 student-facing and teacher-facing surfaces delivered in S1–S7:
- S4: `apps/web/src/app/(teacher)/content/*` (content list, content/new)
- S5: `apps/web/src/app/(student)/practice/page.tsx`,
  `apps/web/src/app/(student)/exam-sim/page.tsx`,
  `apps/web/src/app/(student)/session/[id]/exam/page.tsx`

Checks per surface:
- Typography scale, spacing tokens, shadow tokens — consistent with design system
- Mobile breakpoint pass (student-facing routes at minimum; see Q-1.1-POLISH-3)

### 2c. 5-state matrix audit

Per `UI_CONTRACT.md:547–557` five states required on every data-bound component:
Loading / Empty / Error / Upgrade (402) / Data.

Coverage target: 100% of v1.1 routes. Each gap closes with a regression guard test (+1 test).

### 2d. axe-core spec live-run unlock (ISSUE-0038 carry)

`apps/web/playwright/e2e/exam-content-a11y.spec.ts` and
`student-composer-a11y.spec.ts` are already authored (S4/S5) and `test.skip`-guarded
pending `E2E_WEB_URL`. Gate unlocks when preview environment is provisioned. No code
change required in the polish stage; listed here so the dependency is tracked.

### 2e. State primitive consistency

All loading, empty, error, and upgrade states across v1.1 surfaces must use the shared
primitives from `@mm/ui` (`LoadingState`, `EmptyState`, `ErrorState`, `UpgradeState`).
No inline variants. ISSUE-0047 is a concrete instance; any additional inline definitions
found during the matrix audit fall into this bucket.

### 2f. A11y static pass

Routes in scope (all v1.1 new routes):
- Focus management on route entry (ISSUE-0045 is the concrete gap; verify no others)
- Keyboard navigation reachability for all interactive elements
- Semantic HTML: heading hierarchy, landmark regions, list structures
- ARIA labels on icon-only controls
- `role="alert"` misuse (ISSUE-0046 is the concrete gap; scan all v1.1 new JSX)

## 3. Out of Scope

- **New feature work.** Any feature request surfaced during polish → file Q-1.1-POLISH-*
  for operator round-trip; do not implement.
- **Design system or token overhaul.** Consistency pass uses existing tokens; does not
  introduce a new design system or token file (see Q-1.1-POLISH-1).
- **Performance optimisation beyond audit.** ISSUE-0040 and ISSUE-0041 are in scope
  because they are filed audit findings. No additional perf work absent a filed issue.
- **E2E test authoring.** Playwright specs for new polish-stage flows are deferred to
  the preview deploy gate. Axe-core live-run unlock (ISSUE-0038) is tracking-only here.
- **Production deploy infra.** Deploy preparation is the successor stage.
- **Post-launch backend carries.** ISSUE-0049 (fuzzy dedup) and ISSUE-0050 (cross-import
  dedup) are backend-only, post-launch. Not touched.
- **v1.0 issue remediation.** ISSUE-0009 through ISSUE-0038 (pre-v1.1 carries) are not
  in scope unless operator explicitly pulls one in with a filed rationale.

## 4. Discipline Binding

T1–T5 active per `CLAUDE.md §T-Discipline`.

**T5 gate model.**
- Per-surface batch (student pages, teacher pages, admin surfaces): T5 three-gate
  (mockup sketch → operator approval → layout pass skeleton checkpoint → data-wiring).
- Trivial isolated fix (single-file, clear from issue text, no layout change):
  operator may approve direct fix-and-verify without a mockup gate. Operator decides
  per issue at sketch review.

**Commit discipline.**
- Atomic commit-and-push per DEV-20260515-2.
- Batching strategy TBD at operator discretion per Q-1.1-POLISH-2:
  one commit per ISSUE-NNNN, or batched per cluster (e.g. all a11y static fixes in one commit).

**Other binding rules.**
- Pre-commit secret guard active (`.githooks/pre-commit`; `git config core.hooksPath .githooks`).
- Co-Authored-By absent in all commit messages (BUILD_CONTRACT §11.2).
- No `any` in production code; Zod at all new API boundaries.
- Conventional Commits with scope: `fix(web):`, `fix(api):`, `fix(a11y):`, `chore(dev-context):`.

## 6. Exit Criteria

1. **High:** All High issues closed. (ISSUE-0042 already closed. 0 open High at stage open
   — condition satisfied at start of stage; re-verify no new High filed at S7 close.)
2. **Medium:** All Medium findings from ADR-0040 scope (ISSUE-0039, 0040, 0041, 0043, 0045)
   closed. Any Medium that cannot be closed must be re-classified Low with written
   justification filed in OPEN_ISSUES.md.
3. **Low:** All Low findings from ADR-0040 scope (ISSUE-0044, 0046, 0047, 0048) triaged.
   Each is either closed or carried to v1.2 with an explicit defer note in OPEN_ISSUES.md.
4. **5-state matrix:** 100% coverage of v1.1 surfaces (S1–S7 routes). All gaps closed
   with regression guard tests.
5. **axe-core live-run:** 0 serious/critical violations on all v1.1 routes. Gate is
   ISSUE-0038 (requires preview env). Stage does not close until preview env is available
   and first green run is confirmed — or operator explicitly defers ISSUE-0038 to the
   preview deploy gate stage with written rationale.
6. **Visual consistency:** pass operator review at T5 gate per surface batch. No unresolved
   review items at stage close.
7. **Quality gates:** lint ✅ · typecheck --force (17/17) ✅ · pnpm -r run test (full,
   not tail-truncated) ✅.
