  load test. Do NOT invent numbers. Rows remain `n/a` until measured against
  a real backend.
- **ADR-0027** governs replay determinism: no `Math.random()`, no
  `Date.now()` as algorithm input, no non-deterministic iteration order,
  sorted-key serialisation via `canonicalize()` for any hash input.

  every successful respond/resume; client echoes via `X-Session-Lock`.

- **ADR-0029** governs SDK routing: single `MmClient`, service prefix in
  hook path. Any SDK change must preserve this shape.
- **ISSUE-0008 grep-first rule**: audit all 5 dispatchers before any edit.
  Do not fix what wasn't audited. Scope narrows or expands based on grep
  output.

### D6 — ISSUE-0007 (SDK `X-Session-Lock` header plumbing)

**Scope:** `packages/sdk/src/client.ts` + `packages/sdk/src/hooks/session.ts`
+ `packages/sdk/src/__tests__/client.test.ts` (or new `hooks.test.ts`).

**Changes:**

1. `MmClient.request` — add optional `lockToken?: string` to the options
   parameter; write `headers['X-Session-Lock'] = lockToken` when set.

2. `useRecordResponse` — after a successful respond response (200), extract
   `lock_token` from `RecordResponseResponse` and store it in a `useRef`
   within the hook. Pass the stored `lock_token` as `lockToken` in the next
   `MmClient.request` call. Clear on session end.

3. `useCheckpoint` — same pattern: store `lock_token` from last successful
   respond (or the initial `CreateSessionResponse`); pass it on each
   checkpoint call.

4. `useAbandon` — pass the stored `lock_token` (or none if session was never
   started).

5. Contract tests (~5 new tests):
   - `MmClient.request` with `lockToken` writes `X-Session-Lock` header
   - `MmClient.request` without `lockToken` omits the header
   - `useRecordResponse` passes `X-Session-Lock` on second call using token
     from first response
   - `useCheckpoint` passes `X-Session-Lock`
   - `useAbandon` passes `X-Session-Lock`

**ADR-0031 watch:** if component-state storage proves unworkable (e.g.
ref-based registry needed for cross-hook sharing), file ADR-0031 before
proceeding with a different shape.
