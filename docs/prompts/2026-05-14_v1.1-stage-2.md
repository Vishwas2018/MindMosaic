## Context

v1.1-S1 (Question Bank Foundation) closed 2026-05-14 at commit e76dbfc (impl) + c4c868e
(chore close). Branch `v1.1/exam-content` is 3 commits ahead of origin/main (v1.0.0 tag).

Stage 2 delivers the Practice Exam Composer: a backend-only facility that assembles a
free-form, scored, timed, fixed-sequence exam session from the question bank by pathway +
difficulty mix + time limit. This is NOT a new engine — it extends the existing
`createSession` + `selectItems` flow.

**§N trap (resolved — Q-1.1-2.1):** The informal name "practice exam" is dangerous.
Spec §18 Session Modes table (lines 2619–2624) defines `session_mode='practice'` as
SkillEngine (targeted, unscored, immediate-feedback). A composed mock exam is
`mode='exam'` + `engine_type='linear'` — both enum values exist in migration 0001.
session_mode='practice' MUST NOT be used for composed exams.

Architect resolutions (binding, 2026-05-14):
- Q-1.1-2.1: mode='exam' + engine_type='linear'; no new enum; no migration.
- Q-1.1-2.2: Ephemeral composer_params on CreateSessionRequest; no config table; no migration.
- Q-1.1-2.3: Random uniform within difficulty band; deterministic seeded Fisher-Yates;
  no Math.random; seed = session_id.
- Q-1.1-2.4: Student self-serve; existing pathway feature-flag gate; no auth change.
