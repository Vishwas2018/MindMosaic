### supabase/migrations/0015_assignment_cron_functions.sql

Two pg_cron functions:
```sql
-- fn_mark_overdue_assignments() — daily 01:00 UTC
UPDATE assignment_session
SET    status = 'overdue', updated_at = now()
WHERE  status IN ('pending', 'in_progress')
  AND  EXISTS (
    SELECT 1 FROM assignment a
    WHERE  a.id = assignment_id
      AND  a.due_at + interval '24 hours' < now()
  );

-- fn_sync_assignment_completion() — every 5 minutes
UPDATE assignment_session
SET    status = 'completed', completed_at = sr.updated_at, updated_at = now()
FROM   session_record sr
WHERE  assignment_session.session_id = sr.id
  AND  sr.status = 'processed'
  AND  assignment_session.status = 'in_progress';
```

### 9 handler functions in handlers.ts:

1. `createAssignment(body, caller, db)` → `AssignmentDTO`
   - Validate CreateAssignmentRequest (inline validator)
   - INSERT assignment; INSERT assignment_target per target
   - JOIN skill_node.name + user_profile.display_name for response DTO
   - Parse + log Idempotency-Key header; inline comment DEV-20260523-1

2. `getAssignment(assignmentId, caller, db)` → `AssignmentDTO | null`
   - Role-gated: student via fn_my_assignment_ids(); teacher creator; admin
   - 404 if not found or RLS-hidden

3. `updateAssignment(assignmentId, body, caller, db)` → `AssignmentDTO`
   - Creator-only; 422 UNPROCESSABLE if status !== 'draft'

4. `publishAssignment(assignmentId, caller, db)` → `AssignmentDTO`
   - Teacher/admin only
   - Expand assignment_target to distinct students (direct + class expansion via class_student JOIN)
   - INSERT assignment_session per student (status='pending') — ON CONFLICT DO NOTHING (re-publish idempotency)
   - INSERT outbox_event(type='assignment_assigned') per student
   - UPDATE assignment SET status='published', published_at=now()

5. `archiveAssignment(assignmentId, caller, db)` → `AssignmentDTO`
   - Teacher/admin only; 422 if already 'archived'
   - UPDATE assignment SET status='archived', archived_at=now()

6. `getAssignmentsForStudent(studentId, statusFilter, caller, db)` → `StudentAssignmentDTO[]`
   - Full spec §24.8 role matrix (Q-33.4 Option B)
   - JOIN assignment_session on (assignment_id, student_id) → my_status, my_session_id, completed_at

7. `getAssignmentsForClass(classId, caller, db)` → `AssignmentDTO[]`
   - Teacher/admin only
   - JOIN assignment_target WHERE class_id = classId

8. `getAssignmentTracking(assignmentId, caller, db)` → `AssignmentTrackingDTO`
   - Teacher/admin only
   - JOIN assignment_session LEFT JOIN user_profile → targets[]
   - completion_rate = completed_count / total (Q-33.4 self-resolve evidence)

9. `startAssignment(assignmentId, authorizationHeader, idempotencyKey, traceId, db, assessmentSvcUrl)` → `{ sessionId: string; assignmentSessionStatus: string }`
   - Student only
   - Check assignment published; check assignment_session.status = 'pending' (not yet started)
   - Forward Authorization header to assessment-svc POST /sessions/create with assignment_id
   - On success: UPDATE assignment_session SET status='in_progress', session_id=<new_session_id>
   - Parse + log Idempotency-Key; inline comment DEV-20260523-1
