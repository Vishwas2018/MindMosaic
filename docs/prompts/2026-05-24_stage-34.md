## Constraints

1. **Notification table ‚Äî no new DDL.** Table exists at 0007_new_domains.sql:239.
   Migration 0016 amends fn_drain_outbox_batch only.

2. **Migration 0016 (REQUIRED ‚Äî BLOCKER).**
   - OR REPLACE fn_drain_outbox_batch.
   - Replace `ELSIF event.event_type = 'assignment.published'` with
     `ELSIF event.event_type = 'assignment_assigned'`.
   - Job payload = `event.payload` (JSONB, contains student_id + tenant_id).
   - Add `ELSIF event.event_type = 'plan_updated'` ‚Üí `notification.create`,
     priority `medium`, payload = `event.payload`.
   - Add `ELSIF event.event_type = 'intervention_alert'` ‚Üí `notification.create`,
     priority `medium`, payload = `event.payload`.
   - `assignment.published` branch removed entirely (was dead code).
   - Privilege hardening: triple REVOKE + GRANT service_role (per 0010 pattern).
   - OR REPLACE is idempotent; cron re-schedule uses unschedule-first pattern
     (ADR-0017) for the outbox.dispatch cron.

3. **jobs-worker route map amendment.**
   - Add `notification.create` entry to `buildRouteMap()` in
     `supabase/functions/jobs-worker/index.ts`.
   - Add `NOTIFICATIONS_SVC_URL` env var (default:
     `${SUPABASE_URL}/functions/v1/notifications-svc`).
   - ADR-0031 fourth amendment already filed (docs/dev/decisions/0031-*.md).

4. **orchestration-svc: outbox_event INSERT on replan completion.**
   - At the completion of `processOrchestratorReplan` (after plan_revision INSERT
     and learning_plan UPDATE), add an outbox_event INSERT:
     `event_type = 'plan_updated'`, payload =
     `{ student_id, tenant_id, plan_id, session_count }`.
   - 2-line INSERT at the natural code-path site; no new function.
   - This is a cross-service amendment to shipped Stage 31 code.

5. **analytics-svc: outbox_event INSERT after each intervention_alert INSERT.**
   - In `processTeacherRefresh`, after each `intervention_alert` INSERT, add
     outbox_event INSERT: `event_type = 'intervention_alert'`, payload =
     `{ student_id, teacher_id, tenant_id, alert_id, alert_type }`.
   - Cross-service amendment to shipped Stage 30 code.

6. **NotificationDTO.read = (read_at !== null).** `metadata` field NOT exposed in
   DTO (internal only). `link` may be null.

7. **No email dispatch. No push payload. In-app only.**

8. **TypeScript strict, no `any`, Zod at API boundary.**
   - `NotificationDTO` Zod schema in `packages/types/src/notification.ts`.
   - Request schemas for PATCH /read and POST /read-all in same file.
   - `Idempotency-Key` on `POST /notifications/read-all` (write endpoint per
     BUILD_CONTRACT ¬ß8). PATCH /notifications/{id}/read is idempotent by design
     (setting read_at to now() twice is safe); Idempotency-Key optional there.

9. **Notification spam guard (ISSUE-0025).**
   - `createNotification` handler deduplicates on
     `(user_id, type, payload->>'aggregate_id')` within a 1-hour window.
   - Implementation: `SELECT id FROM notification WHERE user_id = $1 AND type = $2
     AND metadata->>'aggregate_id' = $3 AND created_at > now() - interval '1 hour'
     LIMIT 1` before INSERT.
   - On dedup hit: return `{ deduped: true, notification: null }` with status 200.
     Log field `deduped: true`. Inline comment: `// ISSUE-0025: soft dedup 1h window`.
   - Store `aggregate_id` from incoming payload in `notification.metadata` so the
     dedup query can reference it.

10. **100-unread cap (spec ¬ß27.3).**
    - After INSERT, run: `SELECT count(*) FROM notification WHERE user_id = $1
      AND read_at IS NULL`.
    - If count > 100: `UPDATE notification SET read_at = now() WHERE id IN
      (SELECT id FROM notification WHERE user_id = $1 AND read_at IS NULL
       ORDER BY created_at ASC LIMIT (count - 100))`.
    - Must be in the same logical flow as the INSERT (not a separate cron).
    - Inline comment: `// spec ¬ß27.3: 100-unread cap`.

11. **Copy constants module: `supabase/functions/notifications-svc/notification-copy.ts`.**
    - Exports `getNotificationCopy(type, payload): { title: string; body: string; link: string | null }`.
    - v1 defaults per three in-scope event types:
      - `assignment_assigned`: title = `'New Assignment'`, body = `'You have a new assignment.'`, link = `'/assignments'`
      - `plan_updated`: title = `'Your Learning Plan Updated'`, body = `'Your weekly learning plan has been updated.'`, link = `'/'`
      - `intervention_alert`: title = `'Student Alert'`, body = `'A student may need attention.'`, link = `'/teacher#alerts'`
    - T3 self-resolve: copy details are implementation detail; defaults documented here.

12. **Bell component: `packages/ui/src/Bell.tsx`.**

    - apps/web layout wiring is Stage 40 ‚Äî **DO NOT wire into apps/web this stage.**
