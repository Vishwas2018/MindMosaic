## Context

Stage 30 — Day 43 — L7 Teacher Intervention Intelligence (Spec §14). 1-day budget. Phase 2, +2 days banked. All prerequisite tables exist: `class_group`, `class_student`, `skill_mastery`, `learning_velocity`, `behaviour_profile`, `student_misconception`, `repair_record`, `intervention_alert`, `cohort_metric_cache`. The owning service for this stage is `analytics-svc` — a new Edge Function built from scratch per arch §4.7 (Q-30.1 → ADR-0033). `analytics-svc` is the authoritative owner of `intervention_alert` and `cohort_metric_cache` (arch §1.2 ownership table). No `analytics-svc` Edge Function exists prior to this stage.

ADR-0032 precedent extended to Stage 30 (Q-30.2): skip `pipeline_event` (no session_id for class-scoped job) AND skip `intelligence_audit_log` (no student_id for class-scoped writes — `student_id NOT NULL` FK constraint analogous to `pipeline_event.session_id`). Observability for L7 lives entirely in downstream artifact writes: `intervention_alert` inserts + `cohort_metric_cache` UPSERT carrying `computed_at` + full group payload. Pattern generalises: any non-session-scoped, non-student-scoped pipeline stage uses its domain artifacts as the observability surface.

Pre-implementation artefacts resolved: Q-30.1 (analytics-svc, ADR-0033), Q-30.2 (audit_log skip, ADR-0032 amended), Q-30.3 (hand-rolled Lloyd's k-means in @mm/engines), Q-30.4 (high-fatigue deferred, ISSUE-0017 filed). ADR-0031 route map amended: `pipeline.teacher_refresh → analytics-svc` replaces speculative `pipeline.l7.* → orchestration-svc`.

## Constraints

- No `analytics-svc` exists; scaffold it as a new workspace package in the same pattern as `jobs-worker` (Stage 28): `package.json`, `tsconfig.json`, `handlers.ts`, `index.ts`, `__tests__/contract.test.ts`.
- `pipeline.teacher_refresh` job type: arch §5.2 idempotency key = `teacher:{class_id}`. Job payload: `{ class_id, skill_id, algorithm_version, trace_id }`.
- L7 algorithm version constant: `L7_ALGORITHM_VERSION = 'L7.v1'` (string literal distinct from L5's `'L5.v1'`).
- k-means: hand-rolled Lloyd's in `packages/engines/src/algorithms/kmeans.ts` (Q-30.3 → Option A). Determinism contract: sort input by student_id ASC before clustering; first k points after sort = initial centroids; iteration cap 20; tie-break assignment by group index ASC. No `Math.random`. Max 4 groups. 5D feature vector per student: `[mastery_level, velocity, top_misconception_confidence (0.0 if none), persistence_score, avg_cognitive_load_comfort]`. Exported from `@mm/engines` barrel. Unit tests in `@mm/engines`: (a) 3-student trivial convergence; (b) tie-break determinism; (c) cap hit returns last iteration state; (d) 1-cluster degenerate case.
- No `pipeline_event` write (ADR-0032: no session_id). No `intelligence_audit_log` write (Q-30.2 / ADR-0032 Stage 30 amendment: no student_id for class-scoped event). A `// ADR-0032:` comment at the non-call site explains why. Observability: `intervention_alert` inserts + `cohort_metric_cache` UPSERT.
- `processTeacherRefresh` implements **5 of 6** §14.2 alert types (Q-30.4 → Option B): `declining_performance`, `persistent_misconception`, `repair_failure`, `low_persistence`, `exceptional_progress`. `high_fatigue` deferred per ISSUE-0017; inline comment `// ISSUE-0017: high_fatigue alert deferred — requires per-session fatigue onset data (last 5 sessions), not available from behaviour_profile rolling average` at the trigger-evaluation site.
- Soft dedup for `intervention_alert` inserts: before each insert, check for an existing active alert of the same type for the same student created within the last 2 hours. Skip if found. Window = `interval '2 hours'`. Per (student_id, alert_type, status='active').
- Sorted outputs for replay determinism (ADR-0027): iterate students sorted by student_id ASC when evaluating trigger rules and inserting alerts.
- GET endpoints: Bearer JWT with role check — teacher sees only own classes (enforce `class_group.teacher_id = caller.userId`); `platform_admin` and `org_admin` bypass. Routes placed before global service-role gate (mirror Stage 29 GET pattern).
- PATCH `/analytics/intervention-alerts/{id}` — in arch §4.7 but NOT in DEV_PLAN Stage 30 deliverables. Out of scope this stage.
- §14.3 Targeted Assignment Generation — out of scope this stage.
- `processing_time_ms` in `cohort_metric_cache.value`: `Math.round(eff.perfNow() - t0)` (mirrors L5 pattern; t0 captured after dedup check but before all DB reads).
- No `// ISSUE-0015:` comment at `cohort_metric_cache` writes — class-level grouping data (cohort_key = `class:{class_id}:{skill_id}`) is semantically correct use of the table; no category mismatch.
