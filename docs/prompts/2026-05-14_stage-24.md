## Constraints

1. **TypeScript strict** — no `any` in production code.
2. **No `any` coercions on `SessionSummaryDTO`** — parse via the existing Zod schema from
   `@mm/types`.
3. **Mode branching on `SessionSummaryDTO.mode`** — never infer mode from `score != null`
   or `raw_score != null`.
4. **Hero ring** — 120px SVG, `stroke-dashoffset` animated over 1000ms on mount; respect
   `prefers-reduced-motion` (skip animation, render final state immediately).
5. **Score copy thresholds** (UI_CONTRACT §9.1): `raw_score >= 80` → "Well done",
   `60–79` → "Good effort", `< 60` → "Keep practising".
6. **Score band colours**: `score_band === 'high'` → `var(--correct-500)`,
   `'medium'` → `var(--warn-500)`, `'low'` → `var(--incorrect-500)`. If no `score_band`,
   derive from `raw_score`: ≥80 → `correct-500`, ≥60 → `warn-500`, else `incorrect-500`.
7. **Repair mode** — `mode === 'repair'` is a v1.1 stub: render a `<main>` with
   "Repair session results — available in a future release." and a Back link. No ring, no
   stats. Matches `// PHASE-2:` pattern from BUILD_CONTRACT.
8. **Print-safe** — manual `@media print` sweep at close: hide navigation, hero ring
   animation stops, text-only fallback for dynamic data. Manual sweep per UI_CONTRACT §5.2.
9. **No new deps** — no new npm packages. Recharts already in the workspace if chart
   primitives are needed for the stub skeletons.
10. **`'use client'`** on the Results page (uses `useState`, `useEffect` for ring animation).
11. **Skip link** as first focusable element (`#main-content` target), matching the exam-engine
    page pattern.
12. **`FocusHeader` side-task** (Q-24.7): lift to `@mm/ui` and adopt in Practice page only
    if the main Results screen scope (hero ring, 3-mode variants, print styles, e2e spec)
    closes first. Drop if it risks the 1-day budget.
13. **Atomic commit** — one `feat(web):` commit at stage close per BUILD_CONTRACT §11.1.
