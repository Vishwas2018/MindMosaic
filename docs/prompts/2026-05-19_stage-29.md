## Constraints

- 1-day budget; no new Edge Function service (analytics-svc is a future stage).
- algorithm_version = `'L5.v1'`.
- Replay determinism (ADR-0027): no `Math.random()`, `Date.now()` as formula input;
  sorted iteration order on all collections.
- Idempotency: audit-log dedup on `(student_id, pathway_slug, algorithm_version)`.
  If a row with these three values exists in `intelligence_audit_log`, return 200
  `already_processed` without writing. (Different from L3b which deduped on
  `(session_id, algorithm_version)` — L5 is student+pathway-scoped, not per-session.
  Comment in handlers.ts must explain why the dedup key shape differs from L3b.)
- Spec §12.4 guardrails: ≥5 data points (total_attempts across pathway skills) AND
  ≥7 days of history (min `last_attempted_at` to now). Below threshold: return
  `{ status: 'insufficient_data' }` as the `value.status` field inside the cache
  payload. `insufficient_data` is a payload-level field, NOT a top-level HTTP status.
- Cache key scheme (ISSUE-0015 accepted trade-off):
  - `cohort_key` = `student_id::text`
  - `metric_key` = `'readiness:' || pathway_slug`
  - `time_bucket` = `date_trunc('hour', now())::text`
  UPSERT on conflict (replace value + computed_at for the same hour bucket).
  Cache write site must carry inline comment: `// ISSUE-0015: category mismatch
  accepted; v1.1 migrates to dedicated student_prediction_cache table`.
- 1h TTL: cache fresh = `computed_at > now() - interval '1 hour'`.
- retention_half_life: read from `HALF_LIFE_DAYS_BY_YEAR_LEVEL` keyed by
  `user_profile.year_level` (y5→60, y7→90, y9→120, default→90). New file:
  `packages/engines/src/constants/retention.ts`.
- payload.exam_date: when null, skip the `days_remaining` projection; set
  `projected_readiness: null` and `on_track: null` in the response payload.
  Job payload TS contract: `{ student_id: string; pathway_slug: string;
  exam_date?: string | null }` from day one.
- GET endpoint response envelope (three cases):
  - Cache miss → `200 { status: 'no_data', stale_since: null, payload: null }`
  - Cache expired (`computed_at < now() - 1h`) → `200 { status: 'stale',
    stale_since: <computed_at ISO string>, payload: <last computed value> }`
  - Cache fresh → `200 { status: 'fresh', stale_since: null, payload: <value> }`
- Comment discipline:
  - Cache write site: `// ISSUE-0015` comment (see above).
  - Audit-log dedup shape comment: explain that the dedup key is
    `(student_id, pathway_slug, algorithm_version)` (not session-scoped like L3b)
    because L5 operates at the student+pathway level, not per-session.

- New test baseline: 412 + new tests. Capture full `pnpm -r run test` output

### D1 — `packages/engines/src/constants/retention.ts` (new)

```typescript
export const HALF_LIFE_DAYS_BY_YEAR_LEVEL: Record<string, number> = {
  y5: 60,
  y7: 90,
  y9: 120,
  default: 90,
};

export function retentionHalfLifeDays(yearLevel: number): number {
  const key = `y${yearLevel}`;
  return HALF_LIFE_DAYS_BY_YEAR_LEVEL[key] ?? HALF_LIFE_DAYS_BY_YEAR_LEVEL.default;
}
```

### D2 — `supabase/functions/intelligence-svc/handlers.ts` (extend)

New exports:

**`processPredictiveRefresh(input: PredictiveRefreshInput, db): Promise<PredictiveRefreshResult>`**

Steps:
1. Idempotency check: SELECT intelligence_audit_log WHERE student_id + pathway_slug +
   algorithm_version = 'L5.v1' AND event_type = 'L5.predictive.refresh'. If exists →
   return `{ already_processed: true }`.
2. Load pathway → framework_config (for blueprint jsonb, scoring_rules).
3. Load skill_mastery rows for student across all pathway skills
   (join skill_node to pathway via content graph).
4. Load learning_velocity rows for same skills.
5. Load user_profile.year_level → retentionHalfLifeDays().
6. Spec §12.4 data-threshold guard:
   - total_attempts sum across pathway skills ≥ 5
   - earliest last_attempted_at ≥ 7 days ago
   If fails → write `{ status: 'insufficient_data' }` to cohort_metric_cache + write
   pipeline_event step=5 completed + audit log. Return early.
7. Compute `computePathwayReadiness()` (inline §10.3 weighted sum):
   `sum(mastery_level × blueprint_weight[skill_id])` over all pathway skills.
   Blueprint weights read from `framework_config.blueprint` jsonb.
8. If `exam_date` provided in payload:
   - Parse to Date; compute `days_remaining`.
   - For each pathway skill: `projected = min(1.0, mastery + velocity × days_remaining)`.
   - Apply retention decay: `projected × exp(-0.693 × days_remaining / halfLifeDays)`.
   - `projected_readiness = sum(projected × blueprint_weight[skill_id])`.
   - `on_track = projected_readiness >= target_threshold` (read from
     `framework_config.scoring_rules.target_threshold` or default 0.7).
   If `exam_date` null: `projected_readiness = null`, `on_track = null`.
9. Identify gap skills: skills where `mastery_level < skill_threshold` (default 0.6).
10. For each gap skill: compute `predictMasteryDate()` per spec §12.3:
    `days_needed = (target - current) / velocity` if velocity > 0, else null.
    Adjusted: `days_needed × (1 + 0.5 × current)`.
11. Build result payload (sorted by skill_id ASC for replay determinism):
    ```
    {
      status: 'fresh',
      current_readiness_score: number,
      projected_readiness: number | null,
      on_track: boolean | null,
      gap_skills: Array<{ skill_id, mastery_level, velocity,
                          estimated_mastery_date: string | null }>,
      data_points: number,         // total_attempts sum
      computed_at: string,         // ISO timestamp
    }
    ```
12. UPSERT cohort_metric_cache (cache key scheme per constraints). // ISSUE-0015 comment.
13. Write pipeline_event step=5: pending → processing → completed.
14. INSERT intelligence_audit_log (event_type='L5.predictive.refresh',
    student_id, algorithm_version='L5.v1', input_snapshot={student_id, pathway_slug,
    skill_count, data_points}, output={current_readiness_score, gap_skill_count}).

**`PredictiveRefreshInput` type:**
```typescript
type PredictiveRefreshInput = {
  student_id: string;
  pathway_slug: string;
  exam_date?: string | null;
  tenant_id: string;
  trace_id: string;
};
```
